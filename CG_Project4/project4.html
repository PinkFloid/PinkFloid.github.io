Cloth Simulation and Shading


Overview

This assignment focused on two major parts of computer graphics: physically based cloth simulation and real-time shader-based rendering. In the first half, I implemented a mass-spring cloth model, where a sheet of cloth is represented as a grid of point masses connected by springs. In the second half, I worked with GLSL shaders to implement several visual effects, including texture mapping, bump mapping, displacement mapping, and environment-mapped reflections.

In the cloth simulation part, I first built the cloth grid by generating evenly spaced point masses and connecting them using structural, shearing, and bending springs. These constraints allow the cloth to resist different types of deformation, such as stretching, diagonal shearing, and bending. I then implemented the simulation step by accumulating external forces, applying spring forces, using numerical integration to update point mass positions, and enforcing position constraints to keep the cloth behavior stable.

I also extended the cloth simulator to handle collisions and self-collisions. For collisions with other objects, I implemented responses for spheres and planes so that the cloth could interact with the surrounding scene. For self-collision, I used spatial hashing to group nearby point masses and correct positions when masses became too close to each other. These additions made the cloth behave more realistically by preventing it from passing through objects or intersecting with itself.

In the shader part, I implemented different fragment and vertex shader effects to control how surfaces appear during rendering. Texture mapping used UV coordinates to sample colors from an image texture. Bump mapping used a height map to perturb surface normals, creating the illusion of small geometric detail without actually changing the mesh. Displacement mapping extended this idea by moving vertex positions according to the height map, causing the actual geometry to change. Finally, environment-mapped reflection used a cubemap to approximate mirror-like reflections from the surrounding environment.

Overall, this assignment tied together simulation and rendering. The cloth simulation portion showed how physical behavior can be modeled through point masses, springs, constraints, and collision handling, while the shader portion showed how surface appearance can be controlled through texture sampling, normal perturbation, geometric displacement, and reflection. Together, these parts demonstrated how realistic computer graphics depend on both the physical motion of objects and the visual models used to render their materials.

Task 1.3: Area-Weighted Vertex Normals

For sphere collision, I first check whether the point mass is inside the sphere by comparing the distance from the point mass to the sphere center with the sphere radius. If the distance is smaller than the radius, the point mass has penetrated the sphere. I then compute the closest point on the sphere surface along the direction from the sphere center to the point mass. This gives the target tangent point on the sphere surface. The correction vector moves the point mass from its previous position toward this surface point, scaled by 1 - friction so that friction reduces the amount of movement along the correction direction.

For plane collision, I determine whether a point mass crosses the plane during the current time step. I compute the signed distance of both the previous position and the current position to the plane using the dot product with the plane normal. If the two signed distances have opposite signs, then the point mass has moved from one side of the plane to the other, so a collision occurred.

After detecting a crossing, I compute the intersection point between the point mass trajectory and the plane. This is done by solving for the interpolation parameter along the movement direction from the previous position to the current position. I then move the point mass to a corrected position slightly offset from the plane surface by SURFACE_OFFSET, choosing the offset direction based on which side of the plane the point mass came from. Finally, I apply friction by scaling the correction vector by 1 - friction.

Task 1.4: Handling self-collisions

Self-Collision Implementation

For self-collision, I used spatial hashing to avoid checking every pair of point masses directly. I first divided the space into 3D grid cells and used a hash function to assign each point mass to a cell based on its current position. Then I built a spatial map where each hash value stores the point masses that fall into the same grid cell.

When handling self-collision for a point mass, I look up the point masses in the same spatial cell and check their distances from the current point mass. If another point mass is closer than 2 * thickness, I compute a correction vector that pushes the current point mass away from the other mass until the two masses are separated by at least the allowed thickness distance.

If multiple nearby point masses cause collisions, I average all of their correction vectors. Finally, I divide the correction by the number of simulation steps before applying it to the point mass. This keeps the self-collision response stable across smaller simulation substeps and prevents the cloth from correcting too aggressively in a single step.

Part2: Shaders

Shader Program

A shader program is a small program that runs on the GPU to control how geometry is transformed and how pixels are colored. In this assignment, the vertex shader first processes each vertex by transforming its position and passing information such as normals, UV coordinates, and tangents to the next stage. After rasterization, these values are interpolated across the surface, and the fragment shader uses them to compute the final color of each fragment.

The vertex and fragment shaders work together to create lighting and material effects. The vertex shader provides the geometric information needed for rendering, while the fragment shader uses this information along with light positions, camera position, textures, and surface normals to determine how the material should appear. For example, a fragment shader can sample a texture using UV coordinates, compute lighting from a normal vector, or create reflective effects using a cubemap.

Blinn-Phong Shading

The Blinn-Phong shading model combines three main lighting components: ambient, diffuse, and specular lighting. The ambient component gives the object a small amount of base illumination so that surfaces are not completely black. The diffuse component depends on the angle between the light direction and the surface normal, making surfaces brighter when they face the light directly. The specular component creates shiny highlights based on the viewing direction, the light direction, and the surface normal.

This model is not a fully physically accurate lighting model, but it is efficient and produces visually reasonable results. By combining ambient light, diffuse reflection, and specular highlights, Blinn-Phong gives a simple approximation of how light interacts with a surface while remaining fast enough for real-time rendering.

Texture Mapping

For texture mapping, the fragment shader uses the interpolated UV coordinates to sample a 2D texture. The sampled texture color is then used as the fragment color. This allows an image texture to be mapped onto the surface of the cloth or object, making the material appearance more detailed than using a single constant color.



Bump Mapping and Displacement Mapping

Bump mapping uses a height texture to perturb the surface normals in the fragment shader. This changes the way the surface reacts to light, creating the illusion of bumps and surface detail. However, the actual mesh geometry does not change, so the silhouette and vertex positions remain the same.

Displacement mapping also uses a height texture, but it modifies the actual vertex positions in the vertex shader. Instead of only changing the lighting response, displacement mapping changes the real geometry of the surface. As a result, the object can visibly deform, and its silhouette can change depending on the height map.

In my renders, the difference between bump mapping and displacement mapping was clear. Bump mapping mainly affected the lighting on the surface, while displacement mapping produced stronger geometric changes because the vertices were actually moved. With the sand texture I used, the bump effect was relatively subtle, but the displacement result caused the sphere surface to visibly deform.

Effect of Sphere Mesh Coarseness

Changing the sphere mesh coarseness affects how well the geometry can represent displacement. With a coarse sphere mesh, there are fewer vertices available to move, so displacement can look less smooth and the edge or silhouette of the sphere may appear less accurate. With a finer sphere mesh, there are more vertices, so the displaced shape can follow the height texture more smoothly and the geometry appears more correct.

Bump mapping is less sensitive to mesh coarseness because it does not actually move vertices. It only changes normals during shading, so the overall geometry stays the same. Displacement mapping is more affected by mesh resolution because its result depends directly on how many vertices the mesh has. A higher-resolution sphere can show smoother and more detailed displacement, while a lower-resolution sphere may produce rougher or more blocky deformation.

Mirror Shader

The mirror shader computes a reflection direction using the camera direction and the surface normal. It then samples a cubemap texture along that reflected direction. This approximates mirror-like reflection by using the environment map as the reflected scene, making the cloth and sphere appear reflective.
